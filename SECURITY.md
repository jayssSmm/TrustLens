# Security Policy

## Supported Versions

We actively support the latest version of TrustLens. Please keep your installation up to date to ensure you have the latest security patches.

| Version | Supported          |
| ------- | ------------------ |
| 0.3.x | :white_check_mark: Yes |
| 0.2.x | :white_check_mark: Yes |
| < 0.2.0 | :x: No             |

## Reporting a Vulnerability

**Please do not open a public issue for security vulnerabilities.**

We take the security of TrustLens seriously. If you discover a potential security vulnerability, please report it responsibly using one of the following methods:

1. **GitHub Security Advisory:** Submit a [Private Vulnerability Report](https://github.com/TrustLens/TrustLens/security/advisories/new) via the Security tab (if enabled).
2. **Email:** Contact the maintainers directly at `[SHAHID9664@GMAIL.COM]`.

### Please include the following in your report:
*   A detailed description of the vulnerability.
*   Clear, step-by-step instructions to reproduce the issue.
*   An assessment of the potential impact.

## Response Expectations

The TrustLens team is committed to the following response timeline:
*   **Acknowledgment:** Within 48 hours of receipt.
*   **Investigation:** Rapid validation and impact assessment.
*   **Resolution:** We aim to release a fix or mitigation in a timely manner and will keep the reporter updated throughout the process.

Thank you for helping keep TrustLens safe for everyone!
